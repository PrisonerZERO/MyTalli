Sync

Sync worker infrastructure (code-complete)

ShopSyncWorker BackgroundService polls every 5 min, dispatches per platform via IPlatformSyncService, exponential backoff to 24hr cap. EtsySyncService pulls receipts with pagination + inline token refresh. TokenRefreshWorker handles proactive refresh. Incremental sync uses LastSyncDateTime as the floor with 90-day initial backfill. Commands: RefreshShopTokensCommand, UpdateShopSyncStateCommand, UpsertEtsyRevenueCommand. All DI wired. Note: never run against a live Etsy shop yet — see "Sync worker live-run verification" in Next Up.

Apr 24 2026
Sync

Free-tier Etsy shop cap (1-shop gate)

Multi-shop platforms now respect the plan tier. UI replaces "Connect another shop" with a "Upgrade to Pro" gradient CTA when a free-tier user is at the 1-shop cap. EtsyConnect endpoint enforces the same rule server-side — refuses the OAuth launch and redirects to /platforms?error=etsy_plan_limit with an explanatory toast. Pro users (ProductId 1 or 2, Active or Cancelling) are uncapped. Supersedes the originally-planned IsActive gate — blocking the connection is simpler and more honest than letting them connect many and silently syncing one.

Nav

CurrentUserService circuit-scope fix (Platforms page)

PlatformsViewModel now calls CurrentUserService.Set(userId, string.Empty) in OnInitializedAsync, matching the pattern already used by Dashboard / Goals / Manual Entry / Settings / Suggestions / Unsubscribe. Fixes an unhandled AuditResolver exception on Pause/Resume after a forceLoad navigation round-trip (middleware populates the HTTP-scope CurrentUserService, but Blazor components run in the circuit scope — different instance, never populated). Documented as a rule in CLAUDE.md so future ViewModels don't repeat the bug.
