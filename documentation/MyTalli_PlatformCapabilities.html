Overview

Five Platforms, One Dashboard

MyTalli connects to payment platforms via OAuth so users grant read-only access to their revenue data. Each platform varies in data richness, API style, rate limits, and approval requirements. This document maps what each platform can give us and where the gaps are.

Token expiry is a UX problem, not just a technical one. Etsy (90 days) and PayPal (180 days) refresh tokens expire if the user doesn't visit MyTalli. When a connection breaks, the user must re-authorize — and they may not understand why. Build proactive email reminders before expiry and clear "reconnect" flows in the UI.

Revenue Data

What Each Platform Gives Us

The core question: can we show accurate revenue, fees, and net income for each platform?

Stripe

Balance Transactions API — the gold standard
  • Gross amount, net amount, fee amount per transaction
  • Fee breakdown by component (Stripe fee, app fee, tax)
  • Payout schedule and bank transfer tracking
  • Full refund and dispute history
  • Multi-currency with exchange rates
  • Date filtering via Unix timestamps
  • Cursor-based pagination (efficient for large datasets)

Etsy

Receipts + Transactions + Ledger Entries
  • Order totals, item prices, shipping, taxes
  • Etsy fees and billing charges (separate scope)
  • Ledger entries for running account balance
  • Payment details per receipt
  • Refund data via payment records
  • Multi-currency (buyer vs fee currencies)
  • Webhook payloads are lightweight — require follow-up API call
  • Receipts endpoint may enforce 1 req/s/shop in practice

Gumroad

Sales API — simple but limited
  • Sale amount, product, timestamp, buyer email
  • Flat Gumroad fee (10%) — no component breakdown
  • Date filtering (YYYY-MM-DD)
  • Subscription and refund tracking
  • No payout/disbursement API
  • No net amount — must calculate manually
  • Rate limits undocumented — implement adaptive backoff
  • API docs are sparse — rely on community libraries

PayPal

Transaction Search + Balance APIs
  • Transaction amount, fees, status, timestamp
  • Payer info (name, email, country — not anonymized)
  • Balance inquiry (available, withheld, per currency)
  • Refund and chargeback tracking via T-codes
  • Multi-currency with per-currency balances
  • No net amount field — must calculate (gross - fees)
  • 31-day max date range per API call
  • Transaction data delayed 3-72 hours
  • Partner program may be required for third-party access

Shopify

GraphQL Admin API — Orders + Shopify Payments
  • Order totals, subtotals, taxes, shipping, discounts
  • Transaction detail (gateway, kind, status)
  • Multi-currency (shop money + presentment money)
  • Refund tracking with amounts
  • Nested queries — orders + transactions in one call
  • Fee/net data only for Shopify Payments merchants
  • 60-day order limit without read_all_orders approval
  • GraphQL required for new apps (no REST)
  • Expiring offline tokens mandatory April 2026
Data Sync

Real-Time vs Polling

How fresh can we keep the data? Webhooks give real-time updates; polling fills gaps and handles historical backfill.

Platform Webhooks Key Events Polling Needed? Recommended Strategy
Stripe Full catalog charge.succeeded, charge.refunded, payout.paid Initial backfill only Webhooks primary + daily reconciliation poll
Etsy 4 order events order.paid, order.canceled Yes — webhooks are lightweight (URL only, no payload data) Webhooks trigger + API fetch for details + hourly poll
Gumroad Sale + refund sale, refund, subscription_ended Yes — for reconciliation and missed events Webhooks primary + daily reconciliation poll
PayPal Full catalog PAYMENT.CAPTURE.COMPLETED, PAYMENT.CAPTURE.REFUNDED Yes — transaction data delayed 3-72 hours in search API Webhooks for real-time + daily search API backfill
Shopify Full catalog orders/paid, refunds/create, order_transactions/create Periodic reconciliation (webhook delivery not guaranteed) Webhooks primary + daily reconciliation poll

Universal pattern: Every platform should use webhooks for real-time updates plus periodic polling for reconciliation. No webhook delivery is 100% guaranteed — polling catches anything that was missed. Run reconciliation daily during off-peak hours.

PayPal's Partner program requirements for third-party reporting access are ambiguous — contact their partner team early to understand the path. Don't let approval timelines block your launch.

Watch Out

Platform-Specific Gotchas

Landmines that could derail integration work if not planned for upfront.

Stripe — OAuth Direction Unclear

Stripe is steering new platforms away from traditional Connect OAuth toward Stripe Apps (Marketplace). For a read-only aggregation tool, the recommended path may be building a Stripe App rather than using Connect OAuth directly. Clarify with Stripe support before committing to an approach.

Etsy — Commercial Access is a Hard Gate

Without commercial access approval, the app can only serve 4 shops. The review takes 20+ days and Etsy approves at their sole discretion. If denied, there's no workaround. Additionally, buyer_email access requires a separate case-by-case approval.

Gumroad — Platform Stability Risk

Gumroad was open-sourced and the company has undergone significant changes. API documentation is minimal, rate limits are undocumented, and there's no payout API. Long-term API stability is not guaranteed. Build defensively with good error handling and fallbacks.

PayPal — Data Delay + Partner Ambiguity

Transaction data takes 3-72 hours to appear in the Search API — real-time dashboards must use webhooks, not polling. The Partner program requirement for third-party reporting access is the biggest unknown. Contact PayPal's partner team early. Also: 31-day max date range per query means historical backfill requires multiple sequential calls.

Shopify — Fee Data is Payment-Processor Dependent

Transaction fees and net amounts are only available for merchants using Shopify Payments. Merchants using PayPal, Stripe, or other gateways through Shopify will only have gross order totals — no fee breakdowns. MyTalli can only show gross revenue for these merchants. Also: expiring offline tokens become mandatory April 1, 2026.
