Strengths

  • Full data access — nothing hidden
  • Same code as desktop, just responsive
  • Power users on tablets get a usable experience

Weaknesses

  • Two-axis scrolling is a terrible mobile UX
  • Lose row context when scrolled right (platform/date off-screen)
  • Tabs + filters + grid + cards = a LOT of vertical stacking
  • Touch targets are too small for reliable tapping
  • Nobody actually reads a 6-column grid on a phone
Verdict: Technically complete but practically unusable. This is what happens when you "just make it responsive" without thinking about the use case. Users will pinch-zoom, scroll sideways, lose their place, and close the app.
Mobile Treatment B
Desktop Message
Show the summary cards (quick glance numbers) but replace the grid/tabs area with a friendly message directing users to desktop for the full experience. Not a block — you still see your numbers. Just an honest acknowledgment that tables don't work on phones.

Strengths

Weaknesses

Mobile Treatment C
Keyhole View
Purpose-built mobile layout. No grids — instead, stacked summary cards with mini charts, platform breakdown bars, and a "recent activity" feed (single-column, card-based). The user gets meaningful data in a phone-native format. Full tables live on desktop.

Strengths

  • Purpose-built for phones — not a squeezed desktop layout
  • Summary cards + mini charts answer the "how am I doing?" question instantly
  • Platform breakdown bars give relative context at a glance
  • Recent activity feed is a natural phone pattern (like a bank app)
  • Scrolls vertically only — native mobile feel
  • Subtle desktop CTA for full tables — not a roadblock, just a nudge

Weaknesses

  • More work to build — separate mobile layout, not just CSS tweaks
  • Can't search or filter transactions on mobile
  • No expense or payout detail on mobile (only revenue activity feed)
  • Users who want full data on mobile will be frustrated
Verdict: This is the "Keyhole Data" philosophy brought to life. The phone shows you your numbers through a keyhole — enough to know what's going on, not enough to do accounting. The desktop is where you sit down and work. This matches how side-hustle creators actually use their phones: quick check between other things, not a deep dive into transaction details.
