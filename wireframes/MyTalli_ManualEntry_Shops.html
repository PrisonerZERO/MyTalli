Manual Entry — Manual Shops concept

Today Manual Entry assumes one shop. This design lets users group their manual data into multiple "manual shops" (e.g., Weekend Market, Craft Fair Booth, Freelance). The shop lives on app.ShopConnection with Platform = "Manual" — so the existing per-shop ShopConnectionId FK on Revenue / Expense / Payout carries the grouping, and the Dashboard's per-shop breakdowns "just work" with no new plumbing.

Below: three scenes of the Manual Entry page showing how the shop picker behaves, followed by a small schema diagram.

How it maps to the database

Zero new tables. A manual shop is just a ShopConnection row with Platform = "Manual". The existing ShopConnectionId FK on Revenue/Expense/Payout carries the grouping — nothing else changes.

What this unlocks for free
  • • Dashboard per-shop breakdown groups manual shops alongside Etsy/Stripe
  • • Goals can target a specific manual shop (via the existing Platform filter pattern)
  • • Rename once → every row rolls up to the new name
  • • Future: convert a manual shop into a connected one without touching history
